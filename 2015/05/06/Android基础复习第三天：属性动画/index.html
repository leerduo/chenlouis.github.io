本文翻译自Google Android开发文档。(Guide:Property Animation)
属性动画是Android framework非常有用的动画类型，它几乎可以让你为所有的对象添加动画。你可以在整段时间内定义一个动画来改变对象的属性。属性动画在特定的时间内改变对象的属性值(对象的field)。该属性动画允许定义一下：

  • Duration 默认是300ms
  • Time interpolation:时间插值器，一个计算函数
  • Repeat count and behavior:重复次数和表现
  • Animator sets:动画集合
  • Frame refresh delay: 默认是10ms

    该图展示的是非线性的插值器，先加速后减速的过程。
    下面的类图展示了属性动画的框架：

    ValueAnimator对象负责跟踪你的动画的时机，比如动画执行的时间，当前正在进行动画的属性值。
    ValueAnimator对象封装了一个TimeInterpolator,TimeInterpolator负责定义动画的插值器。除此之外，它还封装了TypeEvaluator对象，它定义了怎么计算动画中的属性值。在第二幅图中，TimeInterpolator的值是AccelerateDecelerateInterpolator(先加速后减速),TypeEvaluator的值是IntEvaluator。
    为了启动动画，创建ValueAnimator对象，然后给它开始和结束的属性值、还有动画的时间。在整个动画中，ValueAnimator计算一个从0到1的elapsed fraction—-基于动画的持续时间和消逝的时间。举个简单的例子，在图1中，elapsed fraction在t=10ms的时候的值是0.25，因为整个时间是40ms.
    当ValueAnimator计算完了elapsed fraction之后，调用当前设置的TimeInterpolator,去计算interpolated fraction,interpolated fraction将elapsed fraction映射为新的fraction—-基于设置的time interpolation.举个简单的例子，在图2中，开始动画是缓慢加速的，那么在t=10ms的时候，interpolated fraction差不多是0.15,小于elapsed fraction的值0.25,在图1中，两者的值总是相等的。
    当interpolated fraction计算完了之后，ValueAnimator会去调用TypeEvaluator来计算当前正在执行动画的属性值。这个属性值的计算基于下面的三个值：starting value,ending value,interpolated fraction.举个简单的例子：在图2中，在t=10ms的时候，interpolated fraction的值是0.15，那么此刻的属性值是:0.15*(40-0)或者说是6.

    How Property Animation Differs from View Animation

  • view animation只能为View对象执行动画，而且有限制，只能执行View对象的一部分，比如执行View的scaling和rotation，但是不执行背景颜色。
  • view animation只改变View被draw的地方，而不是View的本身。(后面的这句话没有搞懂)(可能说的是这个局限吧:当你希望动画停止的时候，View的位置就是当前的位置)For instance, if you animated a button to move across the screen, the button draws correctly, but the actual location where you can click the button does not change, so you have to implement your own logic to handle this.
  • 属性动画可以解决掉上面的缺陷，而且属性动画更加的健壮
  • view animation相对更简单
